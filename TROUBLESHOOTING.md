# N-AutoPost - 네이버 뉴스 불러오기 문제 해결 가이드

## 🔑 중요: API 키 설정

**"인기 뉴스 불러오기"가 작동하지 않는 주된 이유는 API 키 설정입니다.**

### 1단계: .env.local 파일 확인

프로젝트 루트 폴더(`n-autopost`)에 `.env.local` 파일을 열고 다음을 확인하세요:

```bash
VITE_GEMINI_API_KEY=실제_API_키
```

**주의사항:**
- 변수 이름이 **정확히 `VITE_GEMINI_API_KEY`** 여야 합니다 (Vite는 `VITE_` 접두사 필요)
- `GEMINI_API_KEY`나 `API_KEY` 등 다른 이름이면 작동하지 않습니다!
- API 키 앞뒤에 공백이나 따옴표가 없어야 합니다

### 2단계: API 키 발급

https://ai.google.dev/ 또는 https://makersuite.google.com/app/apikey 에서 발급

### 3단계: 서버 재시작 (중요!)

`.env.local` 파일을 수정한 후 **반드시 서버를 재시작**해야 합니다:

```bash
# 터미널에서 Ctrl+C로 서버 종료
# 그 다음
pnpm run dev
```

### 4단계: 브라우저 새로고침

`http://localhost:3001` 에서 페이지를 **하드 새로고침** (Ctrl+Shift+R)

---

## 📝 현재 구현 상태

현재 "인기 뉴스 불러오기"는 **AI가 생성한 뉴스**를 보여줍니다.

### 왜 실제 뉴스가 아닌가요?

1. **브라우저 CORS 제한**: 프론트엔드에서 직접 네이버 뉴스를 스크래핑할 수 없음
2. **네이버 API 미등록**: 네이버 개발자센터에 앱을 등록하지 않아 API 키가 없음

### 향후 계획

**Phase 1 (현재)**: AI 생성 뉴스 (UI/UX 테스트용)
**Phase 2 (추후)**: 네이버 Search API 연동 (실제 뉴스)

## 🐛 문제 해결

### 문제: "뉴스를 불러오지 못했습니다" 알림

**해결책:**
1. `.env.local` 파일의 변수 이름이 `VITE_GEMINI_API_KEY`인지 확인
2. API 키가 유효한지 확인
3. 서버 재시작
4. 브라우저 콘솔(F12)에서 에러 메시지 확인

### 문제: 페이지가 흰 화면으로 표시됨

**해결책:**
- 포트 3001로 접속: `http://localhost:3001`
- 포트 3000은 다른 프로젝트가 사용 중

### 문제: API 모델 에러

SDK 버전 문제일 수 있습니다. `package.json` 확인:
```json
"@google/genai": "^1.38.0"
```

## ✅ 정상 작동 확인

1. "인기 뉴스 불러오기" 클릭
2. 로딩 표시 후 뉴스 목록 30개가 표시됨
3. 각 뉴스에 제목, 언론사, 요약이 표시됨

성공하면 브라우저 콘솔에: 
`✅ AI가 생성한 뉴스 30개 (추후 실제 네이버 API로 전환 예정)`
